# 033 - LLM 供应商与模型动态管理实现总结

## 变更摘要

将原本硬编码在代码中的 LLM 供应商和模型预设列表，改为数据库驱动 + 用户可自主管理的方案。新增**模型配置**管理页面，支持增删改查供应商和模型预设，并让 Agent 表单的提供商/模型下拉列表从 API 动态加载。运行时动态解析供应商配置，无需重启即可使用新添加的供应商。

## 关键实现说明

### 1. 数据库设计
- `llm_provider` 表：`key`（唯一标识）、`name`、`base_url`、`api_key`、`is_builtin`、`is_enabled`、`sort_order`
- `llm_model` 表：`provider_id`（FK，级联删除）、`model_id`、`name`、`is_default`、`sort_order`
- 唯一约束：`(provider_id, model_id)` 防止重复模型

### 2. API Key 安全
- 数据库中存储原始密钥（MVP 阶段，本地部署）
- API 返回时自动脱敏：仅显示末 4 位（如 `****cdef`）
- 更新时若提交脱敏值（以 `****` 开头），自动保留原始密钥

### 3. 运行时动态解析
- 新增 `ProviderConfigLookup` 函数类型，注入到每个 LLM Runner
- `clientForInput` 解析顺序：**动态查找（DB）→ 静态 map（环境变量）→ 报错**
- 新增供应商后无需重启服务，下次 Agent 执行时自动使用

### 4. 前端设计
- 模型配置页：可折叠的供应商卡片，展开后展示模型预设表格
- Agent 表单：页面加载时从 `/api/v1/llm-providers?enabled_only=true` 拉取启用的供应商列表
- 向后兼容：API 失败时自动降级到硬编码列表

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/llm-providers` | 创建供应商 |
| GET | `/api/v1/llm-providers` | 列出所有供应商（附模型，支持 `?enabled_only=true`） |
| GET | `/api/v1/llm-providers/{id}` | 获取供应商详情 |
| PUT | `/api/v1/llm-providers/{id}` | 更新供应商 |
| DELETE | `/api/v1/llm-providers/{id}` | 删除供应商（级联删除模型） |
| POST | `/api/v1/llm-providers/{id}/models` | 添加模型预设 |
| GET | `/api/v1/llm-providers/{id}/models` | 列出供应商模型 |
| PUT | `/api/v1/llm-models/{id}` | 更新模型预设 |
| DELETE | `/api/v1/llm-models/{id}` | 删除模型预设 |

## 已知限制

1. API Key 在数据库中为明文存储（MVP 阶段，本地部署环境可接受；生产环境应加密存储）
2. 环境变量配置的 API Key 不会自动同步到数据库种子记录（需用户在管理页面手动填入）
3. 未实现供应商连接测试功能（如发送测试请求验证 API Key 是否有效）
4. 模型列表未实现自动发现（需手动添加模型预设）
