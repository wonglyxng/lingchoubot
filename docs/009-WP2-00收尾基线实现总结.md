## 关键实现说明

### 1. 鉴权中间件（auth.go）
- **静态 API Key 模式**：通过 `API_KEY` 环境变量配置，请求需携带 `Authorization: Bearer <key>`
- **开发模式**：`API_KEY` 为空时自动放行所有请求（开发友好）
- **公开路径豁免**：`/healthz`、`/readyz`、`/api/v1/ping` 无需鉴权
- **身份上下文**：通过 `context.WithValue` 注入 `Identity` 结构体
- **中间件链**：Recovery → Logging → CORS → **Auth** → mux

### 2. 自动化测试基线
- **model 层**：任务状态机 34 个转换测试（覆盖 9 状态 × 合法/非法转换）、JSON Value/Scan 测试、审批状态常量测试
- **middleware 层**：鉴权中间件 7 个测试（公开路径、缺失 Token、无效 Token、有效 Token + 上下文注入、开发模式）
- **gateway 层**：权限匹配 8 个用例（通配符、精确匹配、缺失权限、空能力列表）

### 3. 审批拒绝链路
Demo 脚本新增步骤：
1. 创建审批请求（关联 in_review 状态任务）
2. 拒绝审批，验证任务自动回退 `in_review → revision_required`
3. 验证审批请求状态变为 `rejected`
4. 恢复任务执行 `revision_required → in_progress`
5. Demo 支持 `API_KEY` 环境变量和 `-key` 命令行参数

## 运行方式

### 启动服务
```bash
# 开发模式（不需要 API Key）
docker compose up -d

# 生产模式（需要 API Key）
API_KEY=your-secret-key docker compose up -d
```

### 运行 Demo
```bash
cd backend
# 开发模式
go run cmd/demo/main.go -v
# 带 API Key
go run cmd/demo/main.go -v -key your-secret-key
# 或使用环境变量
API_KEY=your-secret-key go run cmd/demo/main.go -v
```

1. **鉴权方案为静态 API Key**：适合 MVP 阶段，后续需升级为 JWT 或 OAuth2
2. **服务层测试受限**：service 层依赖具体 repository 类型（非接口），无法直接 mock；需要后续提取接口才能写服务层单元测试
3. **WP2-00-02（MinIO 上传）未在本次实现**：需要修改 docker-compose.yml（属于另一个 dirty 文件）
