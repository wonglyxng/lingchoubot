# WP2-05: 细粒度工具权限与鉴权闭环

## 变更摘要

将工具权限从扁平字符串匹配升级为三层权限检查体系：Agent 能力 → 任务契约约束 → 风险等级控制。新增动作级权限格式（`tool.<name>:<action>`）、风险等级（normal/sensitive/critical）、以及 escalated 工具调用状态。任务契约的 `tool_permissions` 字段现已在 Gateway 中强制执行。

## 关键实现说明

### 1. 三层权限检查

Gateway.Execute() 的权限检查顺序：

1. **Agent 能力检查** — 验证 agent.capabilities 是否包含所需权限
2. **任务契约检查** — 当 task_id 存在时，验证 task_contract.tool_permissions 是否允许该工具
3. **风险等级检查** — 对 ActionAwareTool，根据动作风险等级决定放行/拒绝/升级

### 2. 权限匹配层级

支持从粗到细的权限表达：

| 权限格式 | 含义 |
|----------|------|
| `tool.*` | 全部工具全部动作 |
| `tool.<name>` | 该工具全部动作 |
| `tool.<name>:*` | 该工具全部动作（显式通配） |
| `tool.<name>:<action>` | 该工具特定动作 |
| `tool.<name>.elevated` | 该工具的高权限访问（用于敏感操作） |

### 3. 风险等级

| 等级 | 行为 |
|------|------|
| `normal` | 正常执行，标准审计 |
| `sensitive` | 需要 `tool.<name>.elevated` 权限，否则 denied |
| `critical` | 自动 escalated，不执行，记录审计事件 |

### 4. 任务契约工具权限强制执行

当 ToolRequest 包含 TaskID 时，Gateway 会查询最新版本的 TaskContract，从 `tool_permissions` JSONB 数组中验证该工具是否被允许。空列表或无契约 = 无限制（向后兼容）。

### 5. 审计增强

| 事件类型 | 场景 |
|----------|------|
| `tool_call.denied` | Agent 无权限 / 契约不允许 / 敏感操作无提升权限 |
| `tool_call.escalated` | 高危操作需要审批 |
| `tool_call.failed` | 执行失败 |
| `tool_call.completed` | 执行成功 |

denied 事件现在包含 `denied_reason` 字段，记录具体拒绝原因。

## 已知限制

1. **Critical 工具仅标记 escalated，未集成审批流**：escalated 的工具调用仅记录事件，尚未与 approval_request 表联动自动发起审批
2. **任务契约权限仅检查工具名**：尚未支持契约级别的动作限制（如契约只允许 read 不允许 write）
3. **DocGenerator 和 TestRunner 未实现 ActionAwareTool**：这两个简单工具仍使用基础 Tool 接口，所有操作默认 normal 风险
4. **elevated 权限为静态检查**：敏感工具的 elevated 权限来自 agent.capabilities，未设过期时间或动态授权
5. **API_KEY 鉴权仍为静态方案**：HTTP 层鉴权仍使用环境变量 API_KEY，未支持多角色或 token 刷新
