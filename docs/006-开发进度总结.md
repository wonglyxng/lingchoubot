## 4. 当前已实现 API

### 基础设施
- `GET /healthz` — 健康检查
- `GET /readyz` — 数据库就绪检查
- `GET /api/v1/ping` — 连通性探测

### 核心 CRUD（第 2 周）
- Project: POST / GET / GET{id} / PUT{id} / DELETE{id}
- Phase: POST / GET{projectId} / GET{id} / PUT{id} / DELETE{id}
- Agent: POST / GET / GET{id} / PUT{id} / DELETE{id}
- Task: POST / GET / GET{id} / PUT{id} / PATCH{id}/status / DELETE{id}
- Audit: GET /audit-logs

### 契约与分派（第 3 周）
- TaskContract: POST / GET{id} / PUT{id} / GET{taskId}/contracts / GET{taskId}/contracts/latest
- TaskAssignment: POST / GET / GET{id} / PATCH{id}/status
- HandoffSnapshot: POST / GET / GET{id} / GET{taskId}/latest
- OrgTree: GET /agents/org-tree / GET /agents/{id}/subordinates

### 工件与评审（第 4 周）
- Artifact: POST / GET / GET{id} / POST{id}/versions / GET{id}/versions
- ReviewReport: POST / GET / GET{id}

### 审批与审计时间线（第 5 周）
- ApprovalRequest: POST / GET / GET{id} / POST{id}/decide
- ProjectTimeline: GET /projects/{projectId}/timeline
- TaskTimeline: GET /tasks/{taskId}/timeline

### 工作流编排（第 6 周）
- Orchestrator: POST /api/v1/orchestrator/runs（启动工作流）
- Orchestrator: GET /api/v1/orchestrator/runs（查询运行列表）
- Orchestrator: GET /api/v1/orchestrator/runs/{id}（查询运行详情）

### Tool Gateway（第 7 周）
- ToolGateway: POST /api/v1/tools/call（调用工具）
- ToolGateway: GET /api/v1/tools（查询可用工具列表）
- ToolCall: GET /api/v1/tool-calls（查询调用记录）
- ToolCall: GET /api/v1/tool-calls/{id}（查询调用详情）

**总计：** 约 54 个接口端点。

Demo 脚本位于 `backend/cmd/demo/main.go`，通过 HTTP API 完成完整业务链路验证。

```bash
cd backend && go run ./cmd/demo/          # 默认连接 localhost:8080
cd backend && go run ./cmd/demo/ -v       # 详细输出（含请求/响应体）
```
