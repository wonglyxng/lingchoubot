# 022-WP2-07组织角色矩阵与主管路由实现总结

## 1. 变更摘要

实现 `WP2-07` 的核心编码工作，将组织角色矩阵、主管域路由和返工闭环从设计文档落地到代码。

主要变更：

1. **Agent 模型扩展**：新增 `role_code`（业务职责码）、`managed_roles`（管理的下级角色）、`allowed_tools`（允许工具模板）、`risk_level`（风险级别）四个字段。
2. **Task 模型扩展**：新增 `execution_domain`（任务执行域）、`owner_supervisor_id`（责任主管 ID）两个字段。
3. **编排器主管域路由**：`runSupervisor` 不再查找"任意 supervisor"，而是根据任务的 `execution_domain` 路由到对应主管（`DEVELOPMENT_SUPERVISOR` 或 `QA_SUPERVISOR`）。
4. **返工闭环**：`runTaskChain` 在评审打回后会自动循环回到 Worker → Reviewer 链路，最多重试 3 次，并记录审计日志。
5. **Demo 升级**：Agent 注册从 5 个扩展为 7 个（新增 QA 主管、QA 执行者），并使用 `role_code` 和 `managed_roles` 明确组织关系。

## 3. 关键实现说明

### 3.1 双层角色设计（role + role_code）

保留 `role`（pm/supervisor/worker/reviewer）用于运行时大类和 registry 兼容；新增 `role_code`（如 `DEVELOPMENT_SUPERVISOR`）用于主管域路由和组织关系表达。未提供 `role_code` 时，服务层按 `role` 自动推导默认值。

### 3.2 主管域路由

`findSupervisorByDomain(domain)` 根据 `execution_domain` 映射到对应的 `role_code`：
- `development` → `DEVELOPMENT_SUPERVISOR`
- `qa` → `QA_SUPERVISOR`
- `general` → 回退到任意 supervisor

### 3.3 execution_domain 推断

`inferExecutionDomain(task)` 从任务标题和描述中提取关键词推断执行域。如果已显式设置非 `general` 值则直接返回。优先级：qa 关键词 > dev 关键词 > general。

### 3.4 返工闭环

`runTaskChain` 在 reviewer 完成后检查 task 状态：
- 非 `revision_required` → 正常结束
- `revision_required` → 记录审计日志，回到 worker → reviewer 子链路，最多 3 次

### 3.5 scanAgent 统一扫描

Agent repo 中提取 `scanAgent` 函数统一处理列扫描，避免 16 列 Scan 重复出现在每个方法中。

## 6. 已知限制

1. `escalation_policy` 的运行时判断尚未接入，仍只是 JSON 存储。
2. `managed_roles` 在当前实现中只用于展示和 Demo 数据表达，尚未用于主管分派合法性校验。
3. `allowed_tools` 未与工具权限系统整合。
4. 前端页面尚未展示 `role_code`、`execution_domain`、`owner_supervisor_id` 等新字段。
5. 返工闭环是同步的（在同一个 goroutine 里循环），未与 Temporal 步骤持久化深度整合。
6. 域推断仍依赖关键词匹配，后续应由 PM Agent 显式指定 `execution_domain`。
