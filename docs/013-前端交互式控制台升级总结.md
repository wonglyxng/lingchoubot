# 013 - 前端交互式控制台升级总结

## 变更摘要

将前端从纯只读展示升级为交互式 CRUD 控制台。所有核心实体（项目、任务、Agent）现支持通过 UI 表单创建，新增工作流运行可视化页面。同步更新了 Phase 2 后端新增字段（agent_type、specialization、WorkflowRun/Step）在前端的展示。

## 关键实现说明

### FormModal 组件
- 统一的模态框表单容器：标题栏、表单区域、取消/提交按钮
- 支持 Escape 关闭、点击遮罩关闭
- 提供 FormField、inputClass、selectClass、textareaClass 导出供各页面复用
- submitting 状态时按钮禁用＋文案切换

### 项目/任务/Agent 创建流程
- 各页面使用 `useState` 管理表单数据和 submitting 状态
- 创建成功后自动关闭模态框、重置表单、刷新列表
- 任务创建时动态加载项目列表供下拉选择
- Agent 创建支持选择上级 Agent，实现组织树层级关系

### 工作流运行页面
- 运行列表卡片视图，可点击展开查看步骤
- 步骤按 sort_order 排序，显示名称、状态、耗时、错误信息
- 支持"启动运行"操作（选择项目后调用 orchestrator API）
- 步骤状态有颜色区分（completed=绿色、running=蓝色、failed=红色）

### Agent 页面增强
- 组织树每行现展示 agent_type 标签（模拟/LLM/人工）
- 展示 specialization 标签（后端/前端/测试等，通用不显示）

## 运行方式

```bash
# 前端开发模式
cd frontend
npm run dev

# 或生产构建
npm run build && npm start
```

确保后端 API (`http://localhost:8080`) 运行中。

## 测试方式

1. 访问 `http://localhost:3000`
2. 验证左侧导航栏包含 8 个项（含"工作流"）
3. 进入"项目"页，点击"新建项目"，填写表单提交
4. 进入"任务"页，点击"新建任务"，选择项目后提交
5. 进入"Agent"页，点击"新建 Agent"，选择角色/类型/专长后提交
6. 进入"工作流"页，点击"启动运行"，选择项目后启动
7. 验证生产构建：`npx next build` 成功（11 页面全部生成）

## 已知限制

1. 表单仅覆盖创建操作，编辑/删除尚未实现
2. 创建失败时使用 alert() 提示，未做更优雅的错误展示
3. 工作流运行页面无自动刷新（需手动刷新查看运行状态变化）
4. 阶段（Phase）创建表单尚未实现（需在项目详情页中添加）
5. 工件和审批页面仍为只读模式
