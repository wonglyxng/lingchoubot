## 3. 设计原则

- 主键统一使用 UUID（`uuid-ossp` 扩展）
- 时间字段统一使用 `TIMESTAMPTZ`
- 核心状态字段使用 `TEXT + CHECK constraint`
- JSONB 仅用于半结构化元数据，不替代核心关系模型
- 每次 schema 变更必须有对应 migration
- 不允许手工改库而不落 migration

## 5. 当前表结构摘要

### 5.1 schema_info

系统元数据键值表。当前示例：`version` → `0.1.0`。

---

### 5.2 project

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| name | TEXT, NOT NULL | 项目名称 |
| description | TEXT | 描述，默认 `''` |
| status | TEXT + CHECK | `planning` / `active` / `paused` / `completed` / `cancelled` |
| owner_agent_id | UUID, nullable | 项目负责人 Agent（无外键约束，便于先建项目后绑 Agent） |
| metadata | JSONB | 扩展字段，默认 `{}` |
| created_at / updated_at | TIMESTAMPTZ | 时间戳 |

---

### 5.3 project_phase

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| project_id | UUID, FK → project ON DELETE CASCADE | 所属项目 |
| name | TEXT, NOT NULL | 阶段名 |
| description | TEXT | 描述 |
| status | TEXT + CHECK | `pending` / `active` / `completed` / `skipped` |
| sort_order | INT | 排序，默认 0 |
| metadata | JSONB | 默认 `{}` |
| created_at / updated_at | TIMESTAMPTZ | 时间戳 |

索引：`project_id`。

---

### 5.4 agent

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| name | TEXT, NOT NULL | 显示名 |
| role | TEXT + CHECK | `pm` / `supervisor` / `worker` / `reviewer` |
| description | TEXT | 描述 |
| reports_to | UUID, FK → agent(id), nullable | 上级 |
| status | TEXT + CHECK | `active` / `inactive` |
| capabilities | JSONB | 默认 `[]` |
| metadata | JSONB | 默认 `{}` |
| created_at / updated_at | TIMESTAMPTZ | 时间戳 |

索引：`reports_to`。

> **与早期草案差异：** 早期文档草稿中的 `agent_type`、`role_code`、`allowed_tools` 未在本 migration 中实现；当前以 `role` + `capabilities` + `metadata` 表达。

---

### 5.5 task

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| project_id | UUID, FK → project ON DELETE CASCADE | 所属项目 |
| phase_id | UUID, FK → project_phase ON DELETE SET NULL, nullable | 所属阶段 |
| parent_task_id | UUID, FK → task ON DELETE SET NULL, nullable | 父任务 |
| title | TEXT, NOT NULL | 标题 |
| description | TEXT | 描述 |
| status | TEXT + CHECK | 见下表 |
| priority | INT | 默认 0 |
| assignee_id | UUID, FK → agent ON DELETE SET NULL, nullable | 执行者 |
| input_context / output_summary / metadata | JSONB | 默认 `{}` |
| created_at / updated_at | TIMESTAMPTZ | 时间戳 |

**status 取值：** `pending`、`assigned`、`in_progress`、`in_review`、`revision_required`、`completed`、`failed`、`cancelled`、`blocked`。

索引：`project_id`、`phase_id`、`assignee_id`、`status`、`parent_task_id`。

> **与早期草案差异：** 使用 `assignee_id` 而非 `assigned_agent_id`；合法状态转换由应用层 `internal/model/task.go` 维护。

---

### 5.6 audit_log

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| actor_type | TEXT, NOT NULL | 如 `user` / `agent` / `system` |
| actor_id | TEXT | 操作者标识 |
| event_type | TEXT, NOT NULL | 如 `project.created` |
| event_summary | TEXT | 摘要 |
| target_type / target_id | TEXT | 被操作对象类型与 ID（文本存储 UUID） |
| before_state / after_state | JSONB, nullable | 变更前后快照 |
| metadata | JSONB | 默认 `{}` |
| created_at | TIMESTAMPTZ | 写入时间 |

索引：`(target_type, target_id)`、`event_type`、`created_at`。

### 5.7 task_contract

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| task_id | UUID, FK → task ON DELETE CASCADE | 所属任务 |
| version | INT, NOT NULL | 契约版本号，同一任务内递增 |
| scope | TEXT | 本轮范围 |
| non_goals | JSONB | 非目标，默认 `[]` |
| done_definition | JSONB | 完成定义，默认 `[]` |
| verification_plan | JSONB | 验证步骤，默认 `[]` |
| acceptance_criteria | JSONB | 验收标准，默认 `[]` |
| tool_permissions | JSONB | 允许使用的工具，默认 `[]` |
| escalation_policy | JSONB | 升级策略，默认 `{}` |
| metadata | JSONB | 默认 `{}` |
| created_at / updated_at | TIMESTAMPTZ | 时间戳 |

约束：`UNIQUE(task_id, version)`。索引：`task_id`。

---

### 5.8 task_assignment

| 字段 | 类型 | 说明 |

| id | UUID, PK | 主键 |

| metadata | JSONB | 默认 `{}` |
| created_at | TIMESTAMPTZ | 创建时间 |
| completed_at | TIMESTAMPTZ, nullable | 完成时间 |

索引：`task_id`、`agent_id`。

| metadata | JSONB | 默认 `{}` |
| created_at | TIMESTAMPTZ | 创建时间 |

### 5.10 artifact

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| project_id | UUID, FK → project ON DELETE CASCADE | 所属项目 |
| task_id | UUID, FK → task ON DELETE SET NULL, nullable | 关联任务 |
| name | TEXT, NOT NULL | 工件名称 |
| artifact_type | TEXT + CHECK | `prd` / `design` / `api_spec` / `schema_sql` / `source_code` / `test_report` / `deployment_plan` / `release_note` / `other` |
| description | TEXT | 描述 |
| created_by | UUID, FK → agent ON DELETE SET NULL, nullable | 创建者 Agent |
| metadata | JSONB | 默认 `{}` |
| created_at / updated_at | TIMESTAMPTZ | 时间戳 |

索引：`project_id`、`task_id`、`artifact_type`。

---

### 5.11 artifact_version

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| artifact_id | UUID, FK → artifact ON DELETE CASCADE | 所属工件 |
| version | INT, NOT NULL | 版本号，同一工件内自动递增 |
| uri | TEXT | 存储地址（MinIO/S3 URI 或本地路径） |
| content_type | TEXT | MIME 类型 |
| size_bytes | BIGINT | 文件大小 |
| checksum | TEXT | 校验和 |
| change_summary | TEXT | 本版本变更说明 |
| created_by | UUID, FK → agent ON DELETE SET NULL, nullable | 创建者 Agent |
| metadata | JSONB | 默认 `{}` |
| created_at | TIMESTAMPTZ | 创建时间 |

约束：`UNIQUE(artifact_id, version)`。索引：`artifact_id`。

---

### 5.12 review_report

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| task_id | UUID, FK → task ON DELETE CASCADE | 被评审的任务 |
| reviewer_id | UUID, FK → agent | 评审者 Agent |
| artifact_version_id | UUID, FK → artifact_version ON DELETE SET NULL, nullable | 被评审的工件版本 |
| verdict | TEXT + CHECK | `approved` / `rejected` / `needs_revision` |
| summary | TEXT | 评审总结 |
| findings | JSONB | 发现问题列表，默认 `[]` |
| recommendations | JSONB | 改进建议列表，默认 `[]` |
| metadata | JSONB | 默认 `{}` |
| created_at | TIMESTAMPTZ | 创建时间 |

索引：`task_id`、`reviewer_id`。

> **评审打回逻辑：** 当 verdict 为 `rejected` 或 `needs_revision` 时，服务层自动将关联任务状态转为 `revision_required`。

---

### 5.13 approval_request

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| project_id | UUID, FK → project ON DELETE CASCADE | 所属项目 |
| task_id | UUID, FK → task ON DELETE SET NULL, nullable | 关联任务 |
| artifact_id | UUID, FK → artifact ON DELETE SET NULL, nullable | 关联工件 |
| requested_by | UUID, FK → agent | 发起审批的 Agent |
| approver_type | TEXT + CHECK | `user` / `agent` |
| approver_id | TEXT | 审批人标识 |
| title | TEXT, NOT NULL | 审批标题 |
| description | TEXT | 说明 |
| status | TEXT + CHECK | `pending` / `approved` / `rejected` |
| decision_note | TEXT | 审批意见 |
| decided_at | TIMESTAMPTZ, nullable | 决策时间 |
| metadata | JSONB | 默认 `{}` |
| created_at / updated_at | TIMESTAMPTZ | 时间戳 |

索引：`project_id`、`task_id`、`status`。

> **审批回写：** approved 时自动将关联任务推进为 `completed`；rejected 时推为 `revision_required`（若任务当前状态允许转换）。

## 7. Migration 执行方式

### Docker Compose（自动执行）

```bash
docker compose up
# migrate 服务会在 postgres 健康后自动执行 migration
```

### 手动执行

```bash
cd backend
DB_HOST=localhost go run ./cmd/migrate -direction up    # 升级
DB_HOST=localhost go run ./cmd/migrate -direction down   # 回退一步
```
